# 正则消息屏蔽器 插件

![:name](https://count.getloli.com/@astrbot-plugin-regex-blocker?name=astrbot-plugin-regex-blocker&theme=asoul&padding=7&offset=0&align=top&scale=1&pixelated=1&darkmode=auto)

### 基于 AstrBot 的消息屏蔽插件，通过正则表达式匹配屏蔽用户发送给 LLM 的消息

***
> [!note]
> 本插件需要 AstrBot 版本 >= v3.4.0

### 命令
- `/blocker_status`
  - 显示插件当前状态，包括是否启用、屏蔽规则数量、静默模式状态等。

- `/blocker_list`
  - 显示所有屏蔽规则列表。

- `/blocker_test <消息>`
  - 测试指定消息是否会被屏蔽。
  - 示例: `/blocker_test #napcat`

- `/blocker_reload`
  - 重新加载屏蔽规则（修改配置后使用）。

### 功能特性

- ✅ **正则表达式匹配**：支持使用正则表达式灵活定义屏蔽规则。
- ✅ **LLM 请求拦截**：在消息发送给 AI 之前进行拦截，节省 API 调用。
- ✅ **原始消息检测**：自动获取包含命令前缀（`#`、`/` 等）的原始消息进行匹配。
- ✅ **管理员绕过**：管理员发送的消息可绕过屏蔽规则。
- ✅ **静默模式**：可选择是否在被屏蔽时提示用户。
- ✅ **日志记录**：记录被屏蔽的消息详情，便于审计。
- ✅ **热重载**：支持通过指令热重载配置，无需重启 AstrBot。

### 配置项

| 配置项 | 类型 | 默认值 | 说明 |
| ------ | ---- | ------ | ---- |
| `enabled` | boolean | `true` | 启用或禁用插件。 |
| `block_patterns` | list | `["^#napcat", "^#test", "^napcat", "^test"]` | 屏蔽正则表达式列表。 |
| `silent_mode` | boolean | `false` | 开启后屏蔽消息时不发送提示。 |
| `block_message` | string | `"您的消息已被屏蔽，无法发送给 AI。"` | 屏蔽时发送给用户的提示消息。 |
| `admin_bypass` | boolean | `true` | 开启后管理员消息不受屏蔽限制。 |
| `log_blocked` | boolean | `true` | 开启后记录屏蔽日志。 |

### 正则表达式示例

```regex
# 屏蔽以 # 开头的消息
^#

# 屏蔽包含特定关键词的消息
.*关键词.*

# 屏蔽以 / 开头的命令消息
^/

# 屏蔽包含 URL 的消息
https?://.*

# 屏蔽纯数字消息
^\d+$

# 屏蔽特定格式的消息（如 JSON）
^\{.*\}$

# 组合多个条件：屏蔽以 # 或 / 开头的消息
^[#/]
```

#### 基础用法
```

```

#### 查看状态
```

        - 静默模式: 关闭

```

### 工作原理

```
用户消息 → 平台适配器 → AstrBot → [正则屏蔽插件检查]
                                        ↓
                              匹配成功 → 阻止传播 → 不调用 LLM
                              匹配失败 → 继续传播 → 正常调用 LLM
```

插件使用 `event_message_type` 和 `on_llm_request` 钩子在消息处理流程中进行检查：

1. 获取用户发送的消息文本（包含原始前缀）
2. 检查用户是否为管理员（如开启管理员绕过）
3. 遍历所有正则表达式规则进行匹配
4. 匹配成功则调用 `event.stop_event()` 阻止事件传播
5. 根据配置决定是否发送提示消息

### 常见问题

#### Q: 修改配置后需要重启 AstrBot 吗？
A: 不需要。修改配置后，使用 `/blocker_reload` 指令即可热重载配置。

#### Q: 正则表达式写错了怎么办？
A: 错误的正则表达式会被跳过并在日志中记录警告，不会影响其他规则和插件运行。

#### Q: 为什么我的消息没有被屏蔽？
A: 请检查：
1. 插件是否已启用
2. 正则表达式是否正确（可使用 `/blocker_test` 测试）
3. 您是否为管理员且开启了管理员绕过

#### Q: 可以屏蔽图片等非文本消息吗？
A: 目前仅支持屏蔽文本消息。非文本消息的 `message_str` 为空，不会被匹配。
